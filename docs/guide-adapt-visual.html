Markdown 源文件格式

源文件必须遵循以下标记规则：

## 卷01-学而第一                    ← 卷标题（##）
                                      ← 空行分隔
### 001 学而第一                      ← 篇章（### 编号 标题）
> 孔子                                ← 作者（> 开头）
                                      ← 空行分隔
子曰：「学而时习之，不亦说乎？         ← 正文（逐字注音）
有朋自远方来，不亦乐乎？               ← 正文续行
人不知而不愠，不亦君子乎？」
标记含义示例
## 卷/篇标题## 卷01-学而第一
### NNN 标题篇章标题+编号### 001 学而第一
> 作者/出处> 孔子
空行分隔篇与篇之间
其他行正文逐字注音

适配决策树

不同文献的格式不同，需要根据以下问题调整 build.js：

文献是否有
卷/篇划分？
有
保持 ## 解析逻辑
对应 volumes[]
没有
创建一个虚拟卷
所有篇章归入
每篇作者
是否固定？
是（如老子）
写死 poem.author
去掉 > 解析
否（如古文观止）
保持 > 解析
编号格式
是否为 ### NNN？
是
保持正则不变
否（如第一章）
修改正则匹配
自动编号

适配示例速查

《论语》— 20 篇，固定作者孔子

项目调整
分卷保持 20 篇分卷
作者写死 poem.author = '孔子'
页面标题论语 · 注音版
作者前缀〔春秋〕
默认字体宋体或楷体

《道德经》— 81 章，无分卷，固定作者老子

项目调整
分卷创建一个虚拟卷「道德经」
作者写死 poem.author = '老子'
编号修改正则，支持「第一章」格式
页面标题道德经 · 注音版
作者前缀〔春秋〕

《诗经》— 305 篇，按风雅颂分卷

项目调整
分卷国风/小雅/大雅/颂 各为一卷
作者多为「佚名」，保持 > 解析或写死
页面标题诗经 · 注音版
作者前缀〔周〕
注意事项古字较多，pinyin-pro 可能部分无法注音

《大学》《中庸》— 篇幅短，不分卷

项目调整
分卷各创建一个虚拟卷
分篇按传统章节分段（如「大学之道」为一章）
作者写死（大学：曾子，中庸：子思）
作者前缀〔战国〕

cd my-classic
npm install

# 2. 放入源文件
cp 论语.txt docs/论语.md

# 3. 编辑 build.js（改 inputFile 路径 + 解析规则）
# 4. 编辑 src/index.html（改标题 + 作者前缀）
# 5. 构建 & 预览
npm run build
npm run preview

# 6. 确认无误后部署
git add . && git commit -m "init: 论语注音版"
